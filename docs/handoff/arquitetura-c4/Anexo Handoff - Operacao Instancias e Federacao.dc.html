Operação, instâncias & federação

Como o software do Arah sobe, fica disponível e se comunica num modelo multi-instância: cada implementação serve um ou mais territórios e conversa com o Arah Core para federar com as demais. Estende a Especificação Arah (v1.0) e o Adendo de Monetização (A.1).

Versão 1.0 Escopo Control plane, deploy, federação Públicos Backend · DevOps/SRE · Plataforma Idioma pt-BR
lan
Princípio condutor: o território é a fronteira de dados. A federação é opt-in e escopada; o Core coordena identidade, descoberta, versões e o ledger — mas nunca é dono dos dados do território.
