## Objetivo

Visão **leve** no app mobile para implementadores: repasse do mês, alertas de onboarding, atalho ao painel de transparência e saúde básica da rede — sem replicar o cockpit web.

---

## Funcionalidades

1. **Dashboard implementador** — KPIs resumidos (receita, repasse, comércios ativos)
2. **Alertas** — onboarding pendente, payout processado, health degradado
3. **Atalhos** — abrir cockpit web, painel transparência, convidar comércio
4. **Notificações push** — eventos operacionais críticos

---

## Princípio UX

- Mobile-first; informação densa evitada
- Ações complexas redirecionam ao cockpit web
- Mesmo design system e tokens do app morador

---

## Critérios de aceite

- [ ] Papel implementador ativável após verificação (Programa Implementadores)
- [ ] Próximo repasse visível no app
- [ ] Push para payout e alertas de onboarding
- [ ] Deep link para cockpit e transparência
- [ ] Morador comum não vê UI de implementador
