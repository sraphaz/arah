1 · Canal

MCP × REST: cada um no seu papel

REST — ingestão de produção

Padrão /api/<service>/v1/<rpc>, tipado pela OpenAPI (/openapi.yaml). Determinístico, versionado, com batch de até 20 operações (POST /api/batch/v1/execute) e projeções jmespath por operação para reduzir payload. Cada op do batch conta quota individualmente — batch economiza round-trip, não requisição.

MCP — exploração por agentes

Streamable HTTP em worldmonitor.app/mcp; tools/list público, tools/call autenticado. describe_tool + outputSchema permitem parsing tipado. Reservado ao signal-scout (investigação assistida) — cobertura MCP é subconjunto da REST, então produção não depende dele. Erros mapeados: -32001 auth · -32029 rate limit · _budget_exceeded.

Orçamento do MVP (recomendação): plano API (US$ 99,99/mês) por instância. Com batch e cache por freshness, 1.000 req/dia comportam ~15 categorias × 4 territórios em cadência conservadora (clima 15 min, desastres 10 min, demais 1–6 h). O provider_quota_ledger alarma em 80% e projeta o custo; upgrade para Business (300 req/min, 10.000/dia) só com demanda medida. Limites são rígidos: 429 + Retry-After + X-RateLimit-*, nunca cobrança silenciosa. Catálogo vivo: GET /api/product-catalog (público).

9 · Jurídico

O que exige validação antes do piloto real

Consumir a API pública não incorpora código AGPL ao Arah; os clientes finos oficiais (CLI/SDKs) são MIT. Ainda assim, três pontos vão a parecer jurídico antes da Fase C: (1) ToS da API × uso comunitário multi-território numa chave; (2) atribuição e limites de marca ("dados via World Monitor" no bloco de fonte — sem logo sem permissão); (3) licenças dos provedores originais repassados (GDACS, USGS…). Análise completa, LGPD e retenção no documento Agentes & salvaguardas.
